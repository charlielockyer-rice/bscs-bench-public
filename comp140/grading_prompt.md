# COMP 140: Computational Thinking - Grading Instructions

## Course Context

COMP 140 is an introductory computer science course at Rice University focused on computational thinking and problem-solving using Python. Students learn to:

- Break down complex problems into manageable subproblems
- Design algorithms (recipes) to solve problems
- Implement solutions in Python
- Explain their reasoning in clear English

## Grading Philosophy

The written responses in COMP 140 assess students' understanding of algorithmic concepts, not their ability to write Python code. Students should demonstrate:

1. **Conceptual Understanding**: Can they explain WHY an algorithm works, not just HOW?
2. **Clear Communication**: Can they express technical ideas in plain English?
3. **Logical Reasoning**: Do their explanations follow logically?
4. **Completeness**: Have they addressed all parts of the question?

## Evaluation Criteria

### Full Credit (100%)
- Answer is correct and complete
- Explanation is clear and easy to follow
- Demonstrates deep understanding of the concept
- Uses appropriate terminology without over-complicating

### Partial Credit (50-80%)
- Core idea is correct but explanation lacks clarity
- Minor conceptual errors that don't invalidate the main point
- Missing some details but shows understanding
- Answer is mostly correct but incomplete

### Minimal Credit (20-40%)
- Shows some relevant understanding
- Attempts to address the question
- Contains significant conceptual errors
- Very incomplete but not entirely wrong

### No Credit (0%)
- No answer provided
- Answer is completely off-topic
- Demonstrates fundamental misunderstanding
- Copied from assignment without any original thought

## Grading Guidelines

### For Algorithm/Recipe Questions
- Students should describe steps in a logical order
- Using function names from the assignment is encouraged
- Pseudo-code style is acceptable if clear
- No actual Python code should be required

### For Analysis Questions
- Expect students to relate answers to course concepts
- Accept reasonable alternative interpretations if well-justified
- Look for evidence of thinking, not just memorized answers

### For "Why" Questions
- Require more than just restating what happens
- Look for causal reasoning and deeper understanding
- Partial credit for identifying relevant factors

### For Comparison Questions
- Students should identify similarities AND differences
- Look for balanced analysis, not one-sided answers
- Trade-offs should be acknowledged when relevant

## Common Patterns to Watch For

### Good Responses
- Start with a clear thesis or main point
- Support claims with reasoning or examples
- Connect to concepts learned in the module
- Acknowledge limitations or edge cases

### Weak Responses
- Vague or overly general statements
- Circular reasoning (restating the question as answer)
- Missing the core concept while focusing on details
- Copying assignment text without synthesis

## Scoring Notes

- Award partial credit generously for effort and partial understanding
- Do not penalize minor grammatical errors if meaning is clear
- Accept informal language if technically accurate
- Consider the student's perspective - they are learning these concepts for the first time
